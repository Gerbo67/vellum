# Settings {#top}

The configuration options are in the gear icon menu, at the bottom corner of the sidebar, next to the user avatar. Settings are saved per user and persist between sessions.

## Notifications {#notifications}

Enable desktop notifications to receive an alert every time Vellum receives a new email, even if the browser tab is in the background.

To enable them, the browser will ask for permission the first time you click the option. If the operating system or browser blocks permissions, the option will appear as unavailable.

- **Blocked in browser** — permission was denied by the user or blocked by site settings. To re-enable them, the permission must be changed manually in the browser settings.
- **Not supported** — the browser does not implement the notifications API. This occurs in some mobile browsers or unsupported environments.

Notifications show the project name and the subject of the received email.

## Appearance {#appearance}

Controls the color scheme of the Vellum interface.

### Light {#theme-light}

The interface always uses the light theme, regardless of the operating system or browser settings.

### System {#theme-system}

The interface follows the operating system's color preference. If the system is in light mode, Vellum displays in light. If it is in dark mode, Vellum adapts automatically. This is the default option.

### Dark {#theme-dark}

The interface always uses the dark theme, regardless of the system settings.

## Language {#language}

Switches the interface language between Spanish and English. The change applies immediately without needing to reload the page.

The language also affects email analysis: check names, check details, and the score summary are returned in the selected language.
